PX_FOUNDATION_API bool PxSetMutexProtocol ( const int  protocol)

Specify the strategy employed to avoid priority inversion arising mutex locks.

Note
Priority inversion occurs when low priority threads acquire mutex locks that block higher priority threads contending the same mutex lock. Available strategies to avoid priority inversion are as follows: a) the thread holding the mutex lock inherits the highest priority of all threads contending the mutex lock (PTHREAD_PRIO_INHERIT). b) the thread holding the mutex lock runs with a priority assigned to the mutex lock itself (PTHREAD_PRIO_PROTECT). Both of these choices can be expensive with large numbers of threads, leading to significant performance regression. In this case, an option might be to run without any attempts to alleviate priority inversion (PTHREAD_PRIO_NONE). Extreme caution must be observed to avoid deadlocks either through strict design of the threads that can contend a lock or by creating all threads with the same priority.
The default protocol is PTHREAD_PRIO_INHERIT
Parameters
[in]protocolwill be applied to all mutexes subsequently constructed after successfully calling PxSetMutexProtocol(). If an illegal value of protocol is specified then the default value will be subsequently applied. Legal values are PTHREAD_PRIO_NONE, PTHREAD_PRIO_INHERIT. PTHREAD_PRIO_PROTECT is also a legal value but only when then the realtime scheduling class SCHED_FIFO or SCHED_RR.
Returns
True if the specified protocol is legal, false if illegal.
